# Feature Concept Template

## Purpose
- Capture domain background and problem framing that helps readers understand why the feature exists.

## Usage Order
- Read after `04-spec.md` when additional context is needed for implementation or review.

## Intended Audience
- Contributors who need feature rationale and conceptual context.

## Out of Scope
- Acceptance criteria or canonical behavior definitions.
- Step-by-step implementation procedures.
- Change chronology and issue tracking.

## Canonical Scope
- Terminology and domain framing local to this feature.
- Conceptual models that support, but do not define, feature behavior.

## Canonical Links
- Canonical feature behavior and acceptance: `docs/04-features/<feature>/04-spec.md`.
- Governance map and taxonomy: `docs/00-README.md`.
- Documentation workflow and review gates: `docs/08-document-operations.md`.
- Controlled vocabulary: `docs/09-glossary.md`.

## Problem Context
- What operating or user context creates this feature need?

## Domain Model
- What key entities, states, or constraints matter to this feature?

## Assumptions
- What assumptions does this concept rely on?

## Last Updated When
- When feature framing, domain terms, or conceptual constraints materially change.
