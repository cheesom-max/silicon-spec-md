# Feature Reference Template

## Purpose
- Provide stable lookup material for interfaces, fields, invariants, and mappings related to this feature.

## Usage Order
- Read after `04-spec.md` for implementation and review lookups.

## Intended Audience
- Engineers, QA, and reviewers needing precise reference details.

## Out of Scope
- Canonical behavior decisions.
- Procedural guidance.
- Worked scenarios and narratives.

## Canonical Scope
- Structured reference data that supports the feature implementation.
- Definitions and tables needed for consistent execution.

## Canonical Links
- Canonical feature behavior and acceptance: `docs/04-features/<feature>/04-spec.md`.
- System boundaries and constraints: `docs/02-system-architecture.md`.
- Repository rule constraints: `docs/03-engineering-rules.md`.
- Glossary-controlled vocabulary: `docs/09-glossary.md`.

## Interfaces and Contracts
- List interface names, fields, and constraints.

## Data Definitions
- List values, enums, limits, and format notes.

## Invariants
- List must-hold conditions referenced by implementation and review.

## Last Updated When
- When reference data, contracts, or invariants for this feature change.
