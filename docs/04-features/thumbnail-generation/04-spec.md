# 썸네일 생성 스펙

## 목적
- 영상 설명을 바탕으로 텍스트가 없는 유튜브 썸네일 이미지를 생성하는 기능을 정의한다.

## 사용 순서
- 4단계: 생성 기능 구현 전에 읽고 갱신한다.
- 입력 규칙, 크레딧 차감, 결과 저장 방식이 바뀔 때 수정한다.

## 대상 독자
- 썸네일 생성 요청, 작업 상태, 결과 저장, 크레딧 차감을 구현하고 검토하는 사람.

## 범위 밖
- 수정 기능 상세.
- 결제 적립 상세.
- 범용 디자인 편집 기능.

## 정본으로 다루는 범위
- 생성 입력 규칙.
- 생성 요청의 사용자 가시 동작.
- 생성 시 차감 규칙.
- 수용 기준과 검증 계획.

## 마지막 업데이트 시점
- 입력 조건, 출력 조건, 생성 비용, 완료 기준이 바뀔 때.

## 정본 링크
- 제품 의도는 `docs/01-product.md`를 참고한다.
- 아키텍처 맥락은 `docs/02-system-architecture.md`를 참고한다.
- 무문자 정책은 `docs/06-adrs/06-ADR-0002-no-text-output-enforcement.md`를 참고한다.

## 문제
- 사용자는 영상 내용을 설명하는 것만으로 빠르게 썸네일 초안을 얻고 싶다.
- 그러나 생성 결과에 텍스트가 섞이면 이후 문구 작업이 어려워져 제품 핵심 가치가 무너진다.

## 고객 서사
- 사용자는 업로드할 영상의 핵심 내용을 30자 이상 자세히 적는다.
- 생성 버튼을 누르면 현재 잔액이 확인되고, 충분하면 작업이 시작된다.
- 결과가 완료되면 사용자는 텍스트가 없는 1280x720 썸네일 이미지를 확인하고 다음 단계로 넘어간다.

## 범위
- 영상 설명 입력 필드.
- 30자 이상 입력 검증.
- 10 크레딧 차감 후 생성 작업 시작.
- 단일 썸네일 결과 저장.
- 텍스트 검출 게이트 통과 후 완료 처리.

## 비목표
- 한 번의 요청에서 여러 장 동시 생성.
- 비디오 업로드 기반 자동 프레임 추출.
- 템플릿 선택형 생성.

## 사용자 또는 호출자
- 로그인된 사용자.

## 동작
- 사용자는 30자 이상인 영상 설명을 입력해야 생성 요청을 제출할 수 있다.
- 시스템은 요청 수락 시 잔액을 확인하고 10 크레딧 차감 이벤트를 기록한다.
- 생성 작업은 비동기 상태를 가지며, 사용자에게 진행 중 상태를 보여준다.
- 생성 결과는 텍스트 검출을 통과해야만 완료 상태가 된다.
- 텍스트가 검출되면 내부 재시도를 한 번 수행하고, 재시도 후에도 실패하면 작업을 실패 처리하고 10 크레딧을 환불한다.
- 완료된 결과는 사용자의 이력 목록에 저장된다.

## 수용 기준
- 30자 미만 입력은 생성 요청 전에 차단된다.
- 잔액이 10 미만이면 생성 요청은 시작되지 않는다.
- 생성이 성공하면 잔액이 정확히 10 감소하고 결과 이미지가 저장된다.
- 완료 상태 결과물에는 텍스트 검출 실패 이력이 없어야 한다.
- 생성이 최종 실패하면 차감된 10 크레딧이 원장 환불 이벤트로 복구된다.

## 영향 받는 영역
- 생성 입력 UI.
- 생성 작업 상태 저장.
- 크레딧 원장.
- 이미지 저장소.
- 텍스트 검출 가드.

## 로컬 설계 메모
- MVP는 요청당 결과 1장만 반환한다.
- 출력 크기는 유튜브 기본 썸네일 규격인 1280x720으로 고정한다.
- 생성 모델 호출 세부는 서버 어댑터 뒤에 숨기되, 제품 약속은 결과 기준으로 검증한다.

## 실행 메타데이터
- 실행 단위: 문서, 코드, 외부 API 연동, 저장소 정책.
- 전제 조건: 인증된 사용자, 10 이상 잔액, 생성 모델 연동, 텍스트 검출 가드.
- 실행 후 확인 항목: 작업 상태 전이, 차감 이벤트, 결과 저장, 검출 통과 여부.
- 남겨야 할 증거: 성공 작업 1건에 대한 입력, 작업 상태, 차감 이벤트, 저장 자산 메타데이터.

## 위험과 열린 질문
- 설명 품질이 낮으면 시각 품질도 낮아질 수 있지만 MVP에서는 프롬프트 보조 기능을 별도 제공하지 않는다.
- 텍스트 검출 민감도가 너무 높으면 정상 이미지를 과도하게 차단할 수 있다.

## 검증 계획
- 입력 길이 검증을 확인한다.
- 잔액 부족 시 요청이 차단되는지 확인한다.
- 생성 성공 시 10 크레딧 차감과 자산 저장이 함께 일어나는지 확인한다.
- 텍스트 검출 실패 시 재시도 후 환불 처리되는지 확인한다.
