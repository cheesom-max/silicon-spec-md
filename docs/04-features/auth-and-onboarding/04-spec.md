# 가입과 온보딩 스펙

## 목적
- 사용자가 로그인하고 무료 100 크레딧을 받아 첫 썸네일 생성까지 진입하는 기능을 정의한다.

## 사용 순서
- 4단계: 구현 전에 읽고 갱신한다.
- 인증이나 초기 크레딧 정책을 바꿀 때 수정한다.

## 대상 독자
- 가입, 세션, 초기 잔액, 대시보드 진입 흐름을 구현하고 검토하는 사람.

## 범위 밖
- Polar 결제 상세.
- 썸네일 생성 및 수정의 상세 동작.
- 전역 아키텍처 설명.

## 정본으로 다루는 범위
- 가입과 로그인 흐름.
- 무료 100 크레딧 지급 조건.
- 인증 이후 진입 상태.
- 관련 수용 기준.

## 마지막 업데이트 시점
- 인증 방식, 초기 크레딧 지급 규칙, 첫 진입 흐름이 바뀔 때.

## 정본 링크
- 제품 의도는 `docs/01-product.md`를 참고한다.
- 아키텍처 맥락은 `docs/02-system-architecture.md`를 참고한다.
- 전역 규칙은 `docs/03-engineering-rules.md`를 참고한다.

## 문제
- 사용자는 생성 기능을 쓰기 전에 계정을 만들고 자신의 크레딧 잔액을 신뢰할 수 있어야 한다.
- 초기 가치 경험을 빠르게 주기 위해 무료 100 크레딧 지급이 가입 흐름 안에 자연스럽게 포함되어야 한다.

## 고객 서사
- 사용자는 랜딩 화면에서 제품을 보고 바로 로그인한다.
- 로그인 후 대시보드에 진입했을 때 잔액이 100으로 표시되고, 곧바로 생성 입력을 시작할 수 있어야 한다.
- 재로그인이나 세션 갱신만으로 무료 크레딧이 다시 들어오면 안 된다.

## 범위
- 이메일 매직 링크 기반 가입과 로그인.
- 최초 가입 시 무료 100 크레딧 지급.
- 로그인 후 현재 잔액과 최근 생성 목록을 볼 수 있는 기본 진입.
- 매직 링크 보안 조건, 허용 리다이렉트, 세션 무효화 규칙.
- signup abuse 방지용 최소 요청 제한.

## 비목표
- 소셜 로그인.
- 팀 초대와 공유 워크스페이스.
- 복수 역할 관리.

## 사용자 또는 호출자
- 신규 가입 사용자.
- 재방문 로그인 사용자.

## 동작
- 사용자는 이메일 주소를 제출해 매직 링크를 받는다.
- 인증이 완료되면 사용자 프로필이 생성되고 무료 100 크레딧 지급 이벤트가 정확히 한 번 기록된다.
- 로그인 상태에서는 현재 사용자의 잔액과 자신의 생성 이력만 조회할 수 있다.
- 이미 가입한 사용자가 다시 로그인할 때 무료 크레딧은 다시 지급되지 않는다.
- 매직 링크는 짧은 유효시간을 가지며, 허용된 애플리케이션 경로로만 리다이렉트된다.
- 이미 사용했거나 만료된 매직 링크는 재사용할 수 없다.
- 사용자는 명시적으로 로그아웃할 수 있어야 하고, 보안 사고 대응 시 서버 측 세션 무효화가 가능해야 한다.
- 로그인 시도와 가입 시도에는 이메일 기준과 클라이언트 식별자 기준의 최소 요청 제한이 적용된다.

## 수용 기준
- 신규 사용자가 첫 로그인에 성공하면 잔액이 100으로 표시된다.
- 같은 사용자가 이후 다시 로그인해도 추가 무료 크레딧이 지급되지 않는다.
- 로그인하지 않은 사용자는 생성, 수정, 결제 시작 요청을 수행할 수 없다.
- 한 사용자는 자신의 생성 결과와 잔액만 볼 수 있다.
- 허용되지 않은 리다이렉트 대상은 로그인 완료 후 이동 경로로 사용할 수 없다.
- 만료되었거나 이미 사용된 매직 링크는 인증에 사용할 수 없다.
- 짧은 시간 안에 과도한 가입 또는 로그인 시도가 들어오면 요청이 제한된다.

## 영향 받는 영역
- 인증 세션.
- 사용자 프로필 테이블.
- 크레딧 원장.
- 대시보드 진입 화면.

## 로컬 설계 메모
- 무료 지급은 프로필 생성과 분리된 원장 이벤트로 기록한다.
- 무료 지급 여부는 사용자별 단일 정책으로 보장한다.
- 온보딩 완료 화면은 복잡한 튜토리얼 대신 현재 잔액과 첫 행동만 강조한다.

## 실행 메타데이터
- 실행 단위: 문서, 코드, 데이터베이스 정책.
- 전제 조건: Supabase Auth와 사용자 프로필 구조가 준비되어 있어야 한다.
- 실행 후 확인 항목: 신규 사용자 생성 여부, 무료 지급 이벤트 생성 여부, 대시보드 잔액 표기.
- 남겨야 할 증거: 신규 가입 사용자 1건에 대해 프로필 레코드와 무료 지급 원장 이벤트 1건이 연결된 기록.

## 위험과 열린 질문
- 이메일 전달 지연이 있으면 첫 진입 경험이 느려질 수 있다.
- 소셜 로그인이 필요해질 가능성은 있지만 MVP 범위에는 포함하지 않는다.
- 공유 기기나 탈취된 링크 환경에서는 세션 무효화와 링크 재사용 방지 정책이 중요하다.

## 검증 계획
- 신규 가입 시 프로필 생성과 무료 원장 이벤트 1건 생성 여부를 검증한다.
- 재로그인 시 무료 지급 이벤트가 추가되지 않는지 검증한다.
- 인증되지 않은 사용자 요청이 거절되는지 검증한다.
- 허용되지 않은 리다이렉트 파라미터가 차단되는지 검증한다.
- 만료 또는 재사용된 매직 링크가 거절되는지 검증한다.
- 짧은 시간의 반복 가입 시도가 제한되는지 검증한다.
