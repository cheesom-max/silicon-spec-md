# 제품 문서

## 목적
- 유튜브 썸네일 자동 생성 웹 앱의 안정적인 제품 의도를 정의한다.

## 사용 순서
- 1단계: 프로젝트를 시작할 때 가장 먼저 읽고 수정한다.
- 기능 스펙을 작성하기 전에 반드시 먼저 정리한다.

## 대상 독자
- 제품, 디자인, 엔지니어링, 운영 담당자와 AI 에이전트.

- 기능별 로컬 수용 기준.
- 저장소 전반의 코딩 규칙.

## 정본으로 다루는 범위
- 문제 정의.

## 마지막 업데이트 시점
- 제품 방향, 대상 사용자, 상위 목표, 비목표가 바뀔 때.

## 문제
- 유튜브 크리에이터는 영상의 내용을 바탕으로 클릭을 유도하는 썸네일을 빠르게 만들어야 하지만, 썸네일 제작은 반복적이고 시간이 많이 든다.
- 특히 초안 이미지는 빠르게 만들고 싶지만, 문구는 직접 통제하고 싶어 한다. 자동 생성 이미지 안에 텍스트가 섞이면 편집 자유도와 브랜드 일관성이 크게 떨어진다.
- 지금 이 문제를 해결할 가치가 있는 이유는 영상 업로드 빈도가 높은 크리에이터일수록 썸네일 제작 병목이 심하고, 단순한 크레딧 기반 과금 모델은 즉시 수익화와 사용량 제어에 유리하기 때문이다.

## 고객 서사
- 1인 크리에이터는 영상을 완성한 뒤 업로드 직전에 썸네일을 빠르게 만들고 싶어 한다.
- 현재는 직접 디자인 툴을 열어 배경 이미지를 찾거나, 생성형 이미지 툴에 여러 번 프롬프트를 반복하고, 이미지 안에 들어간 원치 않는 텍스트를 다시 지우는 비효율을 겪는다.
- 이상적인 경험은 다음과 같다: 사용자가 영상 내용을 30자 이상으로 자세히 설명한다. 시스템은 텍스트가 없는 썸네일 이미지를 생성한다. 사용자는 생성본을 고르거나 한 번 더 수정 요청한다. 마지막으로 앱 안에서 직접 문구를 얹고 다운로드한다.
- 고객이 절대 포기할 수 없는 가치는 세 가지다. 빠른 초안 생성, 텍스트 없는 이미지 결과, 그리고 크레딧 차감이 명확하고 예측 가능해야 한다는 점이다.

## 사용자
- 주요 사용자: 유튜브 영상을 직접 제작하고 업로드하는 개인 크리에이터.
- 보조 사용자: 소규모 콘텐츠 팀에서 여러 영상의 썸네일 초안을 빠르게 만드는 운영 담당자.

## 목표
- 영상 설명만으로 텍스트 없는 유튜브용 썸네일 초안을 빠르게 생성한다.
- 생성 결과를 다시 수정할 수 있게 해 반복 작업을 줄인다.
- 생성 이후 사용자가 직접 썸네일 문구를 올릴 수 있게 해 최종 통제권을 보장한다.
- 무료 체험과 유료 크레딧 구매를 같은 제품 흐름 안에 자연스럽게 연결한다.
- 크레딧 차감과 환불 규칙을 명확히 해 신뢰를 만든다.

## 비목표
- 범용 이미지 편집기 제공.
- 팀 협업, 코멘트, 승인 워크플로.
- 영상 편집 기능.
- 템플릿 마켓플레이스나 디자인 자산 판매.
- 월 구독 기반 과금.
- 소셜 퍼블리싱 자동화.

## 성공 지표
- 선행 지표
  - 가입 후 첫 썸네일 생성 완료율.
  - 무료 100 크레딧 중 첫 30 크레딧 소진 비율.
  - 생성 성공 후 다운로드 또는 문구 오버레이 저장 비율.
- 후행 지표
  - 유료 크레딧 구매 전환율.
  - 구매 사용자당 월 평균 생성/수정 횟수.
  - 30일 재방문 생성 사용자 비율.
- 가드레일 지표
  - 생성 실패 후 미환불 비율.
  - 텍스트가 포함된 결과가 사용자에게 완료 상태로 노출되는 비율.
  - 잘못된 잔액 표시 또는 중복 차감 비율.

## 우선순위 테마
- 무문자 결과 보장에 가까운 강한 가드레일을 둔다.
- 크레딧 규칙은 단순하고 예측 가능해야 한다.
- 첫 사용자는 빠르게 가치 경험을 해야 한다.
- MVP에서는 적은 기능보다 명확한 완료 경험을 우선한다.

## 제약
- 프런트엔드는 Next.js와 Vercel을 사용한다.
- 백엔드는 Supabase를 사용한다.
- 결제는 Polar를 사용한다.
- 무료 사용자는 최초 가입 시 100 크레딧을 한 번만 받는다.
- 새 썸네일 생성은 10 크레딧을 차감한다.
- 기존 생성본을 AI로 수정할 때마다 10 크레딧을 차감한다.
- 생성 결과 이미지에는 텍스트가 포함되면 안 된다.
- 문구는 생성 이후 사용자가 직접 추가해야 한다.

## 열린 질문
