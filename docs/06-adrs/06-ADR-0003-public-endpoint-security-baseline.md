# ADR-0003: 공개 엔드포인트 보안 기준선

## 목적
- 공개 SaaS 경로에서 인증, 인가, abuse 방지, DDoS 완화, 데이터 격리를 어떤 기준선으로 다룰지 기록한다.

## 사용 순서
- 6단계: 공개 엔드포인트 보안 경계를 고정할 때 읽고 갱신한다.
- 인증, 공개 엔드포인트, rate limit, Storage 접근 기준이 바뀔 때 수정한다.

## 대상 독자
- 인증, 공개 API, Storage, 다운로드, 생성 요청을 설계하고 검토하는 엔지니어와 AI 에이전트.

## 범위 밖
- 특정 벤더의 세부 설정 화면 설명.
- 전체 운영 절차의 상세 단계.
- 제품 기능 우선순위 결정.

## 정본으로 다루는 범위
- 인증 및 세션 최소 보안 조건.
- 인가 및 데이터 격리 기준.
- 공개 엔드포인트 요청 제한 기준.
- 공개 다운로드와 Storage 접근 기준.

## 마지막 업데이트 시점
- 인증 방식, 공개 엔드포인트 구조, 요청 제한 기준, 데이터 격리 방식이 바뀔 때.

## 정본 링크
- 현재 시스템 구조는 `docs/02-system-architecture.md`를 참고한다.
- 저장소 전반 규칙은 `docs/03-engineering-rules.md`를 참고한다.

## 상태
- 수락됨

## 맥락
- 이 제품은 공개 가입 경로와 비용 유발 생성 경로를 동시에 가진다.
- 무료 크레딧과 외부 이미지 생성 비용 때문에 abuse와 layer-7 트래픽 공격이 직접적인 운영 비용으로 이어진다.
- Supabase RLS만으로는 Storage, 세션, 공개 엔드포인트 보호가 완결되지 않는다.

## 결정
- 매직 링크 인증은 허용된 리다이렉트 경로, 짧은 유효 시간, 재사용 불가를 기본 조건으로 한다.
- 리소스 접근은 로그인 여부가 아니라 소유권 기준으로 판정한다. 작업, 자산, 원장, 주문 반영 기록은 모두 사용자와 연결된다.
- 공개 엔드포인트는 가입, 로그인 요청, 생성, 수정, 결제 시작, 웹훅 수신 경로별로 최소 rate limit와 동시성 제한을 가진다.
- 생성 경로는 사용자별 동시 작업 수를 제한하고, 포화 시 큐잉 또는 거절로 degrade 한다.
- Storage는 비공개 기본값을 사용하고, 사용자 다운로드는 제한된 수명의 서명 URL 또는 서버 검증 후 스트리밍으로 제공한다.
- 서비스 롤 키는 운영 복구와 서버 측 고권한 작업에만 사용한다.

## 실행 메타데이터
- 직접 영향 범위: 로그인, 공개 API, 생성 경로, 다운로드 경로, Storage 정책, 운영 모니터링.
- 적용 전 전제 조건: 사용자 식별, 소유권 메타데이터, 요청 제한 계층, 감사 로그가 준비되어 있어야 한다.
- 적용 이후 핵심 신호: rate limit 발동량, 생성 동시성 초과 건수, 서명 검증 실패 건수, 타 사용자 접근 차단 건수.

## 결과
- 긍정적 결과
  - abuse와 비용 증폭 위험을 줄인다.
  - 리소스 소유권이 구현 전부터 명확해진다.
  - Storage와 DB 권한 모델이 어긋날 가능성을 낮춘다.
- 부정적 결과
  - 초기 구현에서 요청 제한과 다운로드 정책 설계가 조금 더 복잡해진다.
  - 너무 보수적인 제한은 초기 사용자 경험을 해칠 수 있다.
- 후속 작업
  - rate limit 기준값은 운영 데이터를 보고 재조정한다.
  - 보안 사고 대응 런북을 운영 신호와 연결한다.

## 고려한 대안
- 공개 엔드포인트 보호를 플랫폼 기본 설정에만 맡기는 방안
  - 빠르지만 제품별 비용 경로와 소유권 모델을 보호하기에 불충분해 채택하지 않았다.
- 인증과 인가를 feature spec 수준에서만 다루는 방안
  - 반복 누락 가능성이 높아 전역 ADR로 끌어올렸다.
