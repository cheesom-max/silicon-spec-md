# 시스템 아키텍처

## 목적
- 썸네일 생성 앱의 안정적인 기술 구조를 설명한다.

## 사용 순서
- 2단계: `docs/01-product.md`를 정리한 뒤 읽고 수정한다.
- 기능 설계를 시작하기 전에 시스템 경계를 먼저 합의한다.

## 대상 독자
- 기능 스펙을 작성하거나 검토하기 전에 아키텍처 맥락이 필요한 엔지니어와 AI 에이전트.

## 범위 밖
- 제품 전략의 우선순위 판단.
- 저장소 전반의 코딩 규칙.
- 단계별 운영 절차.
- 전체 결정 이력의 상세 본문.

## 정본으로 다루는 범위
- 시스템 경계.
- 주요 컴포넌트.
- 데이터 및 제어 흐름.
- 외부 의존성.
- 안정적인 기술 제약.

## 마지막 업데이트 시점
- 주요 의존성, 컴포넌트 경계, 데이터 소유 구조가 바뀔 때.

## 정본 링크
- 제품 의도는 `docs/01-product.md`를 참고한다.
- 저장소 전반의 규칙은 `docs/03-engineering-rules.md`를 참고한다.
- 지속되는 결정은 `docs/06-adrs/`를 참고한다.

## 시스템 맥락
- 이 시스템은 사용자가 영상 설명을 입력하거나 기존 생성본을 선택해 유튜브 썸네일 초안을 만드는 SaaS 웹 앱이다.
- 시스템 경계 안에는 웹 UI, 인증 세션, 크레딧 원장, 생성 작업 관리, 생성 결과 저장, 텍스트 오버레이 편집, 결제 처리 상태 반영이 포함된다.
- 시스템 경계 밖에는 이미지 생성 모델 제공자, Polar 결제 인프라, 이메일 전달 인프라, 브라우저 다운로드 환경이 있다.

## 핵심 컴포넌트
- Next.js App Router 애플리케이션
  - 사용자 UI, 서버 컴포넌트, 서버 액션 또는 라우트 핸들러를 제공한다.
  - 생성 요청, 수정 요청, 결제 시작, 결과 조회, 텍스트 오버레이 저장 요청을 받는다.
- Supabase Auth
  - 이메일 매직 링크 기반 인증과 세션을 담당한다.
- Supabase Postgres
  - 사용자 프로필, 생성 작업, 자산 메타데이터, 크레딧 원장, Polar 주문 반영 이력을 저장한다.
- Supabase Storage
  - 생성 결과 이미지와 사용자가 저장한 최종 썸네일 결과물을 보관한다.
- 썸네일 생성 어댑터
  - 나노바나나 이미지 생성 호출을 서버 측에서 캡슐화한다.
  - 텍스트 금지 프롬프트 규칙과 후처리 가드레일을 적용한다.
- 텍스트 검출 가드
  - 생성 결과에 텍스트가 포함되었는지 검사한다.
  - 검출 실패 시 자동 재시도 또는 실패 환불 흐름으로 넘긴다.
- Polar 결제 연동 계층
  - 구매 요청 시 체크아웃 URL을 생성한다.
  - `order.paid` 기반 웹훅을 받아 크레딧을 부여한다.

## 경계와 인터페이스
- 브라우저는 직접 크레딧을 변경하지 않는다. 모든 차감과 적립은 서버를 통해서만 반영한다.
- Supabase의 공개 키로 접근하는 읽기/쓰기 범위는 RLS 정책으로 제한한다.
- 생성 모델 API 키와 Polar 시크릿은 서버 런타임에서만 접근한다.
- 썸네일 생성 결과의 완료 여부는 이미지 파일 자체가 아니라 작업 상태와 텍스트 검출 결과를 함께 보고 판정한다.
- 서비스 롤 키는 브라우저로 노출하지 않으며, 결제 웹훅 반영, 스토리지 보정, 운영 복구처럼 서버 측 고권한 작업에만 제한적으로 사용한다.

## 보안 경계
- 인증은 Supabase 매직 링크를 사용하되, 링크 만료 시간, 허용 리다이렉트 대상, 세션 무효화 규칙을 별도 보안 요구사항으로 관리한다.
- 인가는 단순 로그인 여부가 아니라 리소스 소유권 기준으로 판정한다. 프로필, 작업, 자산, 최종 산출물, 원장 이벤트, 주문 반영 기록은 모두 사용자 소유자와 연결되어야 한다.
- Storage 접근은 데이터베이스 소유권 정책과 동일한 기준으로 맞춘다. 공개 버킷을 기본값으로 두지 않고, 다운로드가 필요할 때만 제한된 수명의 서명 URL 또는 서버 중계 응답을 사용한다.
- Polar 웹훅은 idempotent 처리만으로 충분하지 않다. 서명 검증과 이벤트 재생 방지, 주문-사용자-SKU 바인딩이 모두 통과해야 적립을 허용한다.
- 공개 엔드포인트는 로그인, 가입, 생성, 수정, 결제 시작, 웹훅 수신 경로별로 별도의 rate limit와 동시성 한도를 가진다.
- 생성 작업은 비용 증폭을 막기 위해 사용자별 동시 실행 수를 제한하고, 큐 포화 시 지연 또는 거절로 degrade 할 수 있어야 한다.

## 데이터 흐름
- 가입 시
  - 사용자가 이메일 매직 링크로 가입한다.
  - 사용자 프로필이 생성되고 무료 100 크레딧 지급 이벤트가 원장에 기록된다.
- 새 썸네일 생성 시
  - 사용자가 30자 이상 설명을 제출한다.
  - 서버가 잔액을 확인하고 10 크레딧 차감 원장 이벤트를 기록한다.
  - 생성 작업이 만들어지고 이미지 생성 어댑터가 실행된다.
  - 결과 이미지는 텍스트 검출을 거친 뒤 통과하면 Storage와 작업 이력에 저장된다.
  - 실패하면 차감 원장을 상쇄하는 환불 이벤트를 기록한다.
- 생성본 수정 시
  - 사용자가 기존 생성본을 선택하고 수정 지시를 제출한다.
  - 서버가 동일하게 10 크레딧을 차감한 뒤 수정 작업을 실행한다.
  - 결과 이미지는 새 자산으로 저장되고 이전 버전과 연결된다.
- 결제 시
  - 사용자가 유료 크레딧 팩을 선택한다.
  - 서버가 Polar 체크아웃 세션을 생성한다.
  - 결제 완료 후 Polar 웹훅이 도착하면 서버가 idempotent 하게 적립 원장을 기록한다.

## 제어 흐름
- 사용자 요청의 주 제어 흐름은 Next.js 서버 계층에서 시작해 Supabase와 외부 서비스로 이어진다.
- 인증이 필요한 모든 쓰기 요청은 현재 세션 사용자 식별 후 처리한다.
- 크레딧 부족, 권한 없음, 텍스트 검출 실패, 웹훅 중복 수신 같은 예외는 서버에서 최종 판정한다.
- 장시간 작업은 비동기 상태를 갖는 생성 작업 레코드로 추적하고, UI는 상태를 조회해 진행 상황을 보여준다.

## 외부 의존성
- Next.js: App Router 기반 웹 애플리케이션 프레임워크.
- Vercel: 애플리케이션 배포와 서버 런타임.
- Supabase Auth: 이메일 매직 링크 인증.
- Supabase Postgres: 원장, 작업, 자산 메타데이터 저장.
- Supabase Storage: 이미지 파일 저장.
- Polar: 일회성 크레딧 팩 판매와 결제 웹훅.
- 나노바나나: 텍스트 없는 썸네일 이미지 생성 모델 제공자.

## 기술 제약
- MVP는 Next.js App Router 기준으로 설계한다.
- 환경 변수와 시크릿은 서버 런타임에서만 읽는다.
- 무료 크레딧과 유료 적립은 모두 원장 이벤트로만 기록한다.
- MVP 출력 형식은 유튜브 기본 썸네일 크기인 1280x720 단일 비율로 제한한다.
- MVP는 다중 팀, 공유 워크스페이스, 배치 생성, 구독제를 지원하지 않는다.
- MVP라도 가입, 생성, 수정, 결제 시작 경로에는 최소한의 abuse 방지 정책과 요청 제한을 둬야 한다.

## 현재 상태의 결정
- 인증은 Supabase 이메일 매직 링크를 기본값으로 사용한다.
- 결제는 구독이 아닌 Polar 기반 일회성 크레딧 팩으로 처리한다.
- 크레딧 잔액은 누적 필드보다 원장 합산을 정본으로 삼는다. 상세는 `docs/06-adrs/06-ADR-0001-credit-ledger-and-deduction-policy.md`를 참고한다.
- 무문자 요구사항은 프롬프트 규칙과 후처리 검출 게이트를 함께 사용한다. 상세는 `docs/06-adrs/06-ADR-0002-no-text-output-enforcement.md`를 참고한다.

## 위험 구간과 핫스팟
- 크레딧 차감, 환불, 결제 적립이 서로 다른 이벤트 경로에서 발생하므로 원장 정합성이 핵심 위험 구간이다.
- 생성 결과에 텍스트가 포함되면 제품 핵심 약속을 깨므로 검출 민감도와 실패 처리 정책이 중요하다.
- 웹훅 중복 수신이나 지연 수신은 중복 적립 또는 미적립으로 이어질 수 있어 idempotency 키 설계가 필수다.
- 사용자 자산 접근은 Storage 정책과 데이터베이스 소유권 정책이 함께 맞물려야 한다.
- 무료 크레딧과 외부 모델 비용이 결합되어 있어 signup farming, generation abuse, layer-7 트래픽 급증이 직접적인 비용 손실로 이어질 수 있다.
- 매직 링크 로그인은 구현 실수 시 open redirect, 세션 고정, 재사용 링크 문제로 이어질 수 있다.
