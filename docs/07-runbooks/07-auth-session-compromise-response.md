# 인증 또는 세션 침해 대응 런북

## 목적
- 매직 링크 오남용, 계정 탈취 의심, 세션 침해 정황이 발생했을 때 반복 가능한 대응 절차를 제공한다.

## 사용 순서
- 7단계: 인증 또는 세션 관련 보안 사고가 발생했을 때 사용한다.
- 인증 방식, 세션 무효화 방법, 로그 구조가 바뀌면 갱신한다.

## 대상 독자
- 인증 관련 보안 사고를 실제로 조사하고 대응해야 하는 운영자와 엔지니어.

## 범위 밖
- 전체 인증 아키텍처 설계.
- 장기 정책 문서.
- 결제 장애 대응.

## 정본으로 다루는 범위
- 트리거 조건.
- 사전 조건.
- 실행 절차.
- 검증, 복구, 에스컬레이션.

## 마지막 업데이트 시점
- 인증 방식, 세션 무효화 절차, 로그 조회 방식이 바뀔 때.

## 정본 링크
- 관련 기능 스펙은 `docs/04-features/auth-and-onboarding/04-spec.md`를 참고한다.
- 관련 ADR은 `docs/06-adrs/06-ADR-0003-public-endpoint-security-baseline.md`를 참고한다.

## 트리거
- 사용자가 본인이 하지 않은 로그인 또는 자산 접근을 신고한 경우.
- 만료되지 않은 매직 링크 재사용, 비정상 리다이렉트, 세션 오남용이 감지된 경우.
- 짧은 시간에 특정 계정 또는 이메일에 로그인 시도가 급증한 경우.

## 사전 조건
- 사용자 ID 또는 이메일 주소를 확보해야 한다.
- 인증 이벤트와 세션 관련 로그 조회 권한이 있어야 한다.

## 필요한 접근 권한과 도구
- 인증 대시보드 또는 세션 관리 도구.
- 애플리케이션 로그 조회 도구.
- 필요 시 세션 무효화 권한.

## 실행 메타데이터
- 적용 범위: 특정 사용자 계정 또는 특정 시간 범위의 인증 이벤트.
- 실행 전 기록할 기준 상태: 사용자 ID, 최근 로그인 시간, 활성 세션 수, 관련 IP 또는 클라이언트 식별자.
- 실행 중 확인할 신호: 만료 링크 사용 여부, 허용되지 않은 리다이렉트 시도, 동일 계정 반복 로그인 시도.

## 절차
1. 사용자 ID 또는 이메일을 기준으로 최근 인증 이벤트와 세션 상태를 조회한다.
2. 비정상 로그인 시도, 재사용 링크, 허용되지 않은 리다이렉트 시도가 있었는지 확인한다.
3. 침해 가능성이 있으면 해당 사용자의 활성 세션을 무효화한다.
4. 동일 시간대의 자산 조회, 생성, 결제 시작 기록을 함께 확인해 2차 영향이 있는지 본다.
5. abuse 패턴이 보이면 해당 식별자에 임시 제한을 적용하고 추가 모니터링을 시작한다.
6. 사용자에게 필요한 경우 재로그인 또는 링크 재발급 안내를 제공한다.
7. 원인과 영향 범위를 기록하고 재발 방지 항목을 문서에 남긴다.

## 검증
- 의심 세션이 더 이상 유효하지 않아야 한다.
- 추가적인 비정상 로그인 또는 자산 접근이 중단되어야 한다.
- 사용자 영향 범위와 대응 내역이 운영 기록에 남아 있어야 한다.

## 롤백 또는 복구
- 잘못 무효화한 세션이 있더라도 보안상 자동 복구하지 않고 정상 로그인 재수행으로 복구한다.
- 구조적 문제라면 인증 경로의 rate limit와 허용 리다이렉트 설정을 우선 점검한다.

## 에스컬레이션
- 동일 계정 또는 동일 패턴의 인증 사고가 반복되면 인증 담당 엔지니어에게 에스컬레이션한다.
- 다수 계정에 영향을 주는 경우 운영 책임자와 보안 책임자에게 즉시 공유한다.

## 운영 피드백 루프
- 사고 유형을 링크 재사용, 리다이렉트 오남용, 세션 탈취 의심 등으로 분류해 남긴다.
- 반복 패턴이 보이면 `docs/04-features/auth-and-onboarding/04-spec.md`와 관련 ADR을 갱신한다.
